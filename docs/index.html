Module ovtable

Fast, friendly, and dynamic ordered table support for your codebase.

Ordered tables from this package work like any other table. Whether you want to use unordered keys, ordered ones, or even numeric indices. It all works fine. That's stellar compatibility, and it's zero-overhead when you're doing native operations.

Need to report an issue? https://github.com/well-in-that-case/ovtable/issues

pkg.orderedtable (override_pairs)
Returns a new orderedtable. Optionally override the __pairs metamethod.

Parameters:

  • override_pairs A boolean indicating whether to overrride the __pairs metamethod.
pkg.orderediterator (...)
Returns an iterator for ordered fields. If you set override_pairs as true, this is the same as pairs. This function takes the same parameters you'd give to __pairs.

Parameters:

  • ...
pkg.add (t, key, value, raw)
Performs t[key] = value & updates the insertion table accordingly. This is the only way to make an ordered key.

Parameters:

  • t The ordered table.
  • key The key to use.
  • value The value to use.
  • raw A boolean indicating whether to use raw assignment (via rawset) or __newindex assignment.

Returns:

    A boolean indicating if the key was added successfully.
pkg.mod (t, key, value)
Modify and reorder this key. Use traditional reassignment if you do not wish to reorder.

Parameters:

  • t The ordered table.
  • key The key to modify.
  • value The new value for this key.
pkg.del (t, ...)
Deletes and removes the insertion table entries for the keys you pass. This function takes an indefinite amount of string arguments. It uses raw assignment to nil values.

Parameters:

  • t The ordered table.
  • ... The keys you wish to delete.

Returns:

    Returns the amount of keys deleted.
pkg.getindex (t, idx, give_key_name)
Gets an ordered field's value by its insertion index.

Parameters:

  • t The ordered table.
  • idx The numeric index to read.
  • give_key_name A boolean indicating whether to also return the key's name.

Returns:

    The key's value or the key's name and value, respectively.
pkg.keyindex (t, key)
Returns the insertion index of the key.

Parameters:

  • t The ordered table.
  • key The key to read.

Returns:

    The key's insertion index (number), or nil if it wasn't found.
pkg.orderedlen (t)
Returns the amount of ordered keys in this ordered table.

Parameters:

  • t The ordered table.

Returns:

    The amount of ordered keys, or nil if this table isn't an ordered table.
pkg.assertioncalls (state)
Whether or not to use assertion calls in these package's functions. Assertion calls can add upwards of 30% overhead during heavy stress. There's no reason to keep using assertion calls if your code is stable & you're familiar with ovtable.

Parameters:

  • state A boolean indicating whether to use assertion calls.

Returns:

    This function has no return.
